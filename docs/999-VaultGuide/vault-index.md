## 📊 Vault Metadata Standards

### **Frontmatter Standardization**

All documents in this vault follow standardized frontmatter conventions documented in **Phase 10** of the refactoring plan.

**Key Metadata Fields**:
- `title`: Human-readable document title
- `document-type`: From controlled vocabulary (see below)
- `project`: Always "Procureline"
- `created`: Document creation date (YYYY-MM-DD)
- `last-updated`: Last modification date (YYYY-MM-DD)
- `status`: Document status (complete, in-progress, archived, deprecated)
- `tags`: Hierarchical tags using `category/subcategory/tag` format

**Controlled Document Types**:
- `screen-design` - Screen design specifications
- `architecture-decision-record` - ADR documents
- `session-log` - BMad session logs
- `pipeline-completion-update` - Pipeline completion updates
- `pipeline-design-plan` - Pipeline design plans
- `research-analysis` - Research documents
- `design-system` - Design system documentation
- `vault-index` - Navigation and indexes
- `project-brief` - Project briefs

### **Tag Taxonomy**

See **[[tag-index]]** for complete tag taxonomy and usage guidelines.

**Tag Categories**:
1. Document Type (`document-type/`)
2. Pipeline (`pipeline/`)
3. Status (`status/`)
4. Quality (`quality/`)
5. Component (`component/`)
6. Impact (`impact/`)
7. Architecture (`architecture/`)
8. Session (`session/`)
9. Design (`design/`)
10. Vault Meta (`vault-meta/`)

### **Metadata Governance**

- **Frontmatter Changes**: Follow standardized schema (see Phase 10)
- **New Tags**: Must be added to [[tag-index]] with controlled vocabulary approval
- **Date Formats**: Always `YYYY-MM-DD`
- **Status Values**: Use controlled vocabulary only
