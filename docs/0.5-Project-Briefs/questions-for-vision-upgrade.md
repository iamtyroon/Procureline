### # Market Growth & Trends

**Q: 10. Is the higher education sector in Kenya/East Africa growing or declining?**

A: **[RESEARCHED ANSWER]:** Growing. University student enrollment in Kenya doubled to surpass 600,000 mark in 2024. Government allocated KES 84 billion (US$615.8 million) for universities in 2023-24 fiscal year, up from KES 54 billion previously. However, universities face financial challenges with Sh14 billion lost through corruption/mismanagement, indicating need for better procurement controls.

**Q: 11. Are government procurement budgets increasing or decreasing?**

A: **[RESEARCHED ANSWER]:** Mixed picture - total allocations increasing but most goes to salaries (94% in some cases), leaving little for development. This creates pressure for efficient procurement processes to maximize limited budgets. Financial audits showing Sh14B in losses indicate scrutiny and demand for transparency is increasing.

**Q: 12. Are there regulatory trends pushing toward digitization/transparency in procurement?**

A: **[RESEARCHED ANSWER]:** Yes, strong push toward digitization:
   - Kenya's IFMIS system mandated for all public universities
   - Uganda launched e-GP system in July 2021
   - Public Procurement and Asset Disposal Act (PPDA) 2015 emphasizes transparency and efficiency
   - Kenya Data Protection Act 2019 requires proper data handling (modeled after GDPR)
   - Government audits increasingly focusing on procurement compliance

**Q: 13. What is driving demand for procurement software in your target market?**

A: **[RESEARCHED ANSWER]:** Key drivers:
   - Mandatory government e-procurement compliance (IFMIS, e-GP)
   - Corruption reduction pressure (Sh14B losses documented)
   - Budget constraints requiring efficiency
   - Transparency and accountability requirements
   - Complexity of multi-departmental coordination
   - Digital transformation trends across education sector

**Q: 14. Are there any government initiatives supporting procurement digitization?**

A: **[RESEARCHED ANSWER]:** Yes:
   - Kenya's IFMIS (Integrated Financial Management Information System) - mandatory for public institutions
   - Uganda's e-GP (Electronic Government Procurement) system - rolled out 2021
   - Tanzania, Rwanda, Ethiopia have similar e-government procurement portals
   - Public Procurement and Asset Disposal Act 2015 (Kenya) mandates electronic procurement processes

**Q: 224. What is your multi-tenancy approach?**

A: - Shared database with tenant ID?
   - Separate database per university?

   **[VAULT ANSWER]:** Shared database with Row-Level Security (RLS) and tenant_id isolation:
   - **Architecture**: Single PostgreSQL database with tenant_id column in all tables
   - **Security**: PostgreSQL Row-Level Security (RLS) policies enforce tenant isolation at database level
   - **Session Management**: 8-hour sessions recommended for administrative workflows
   - **Validation**: Multi-tenant isolation architecture validated as production-ready with 9/10 implementation confidence
   - **Benefits**: Cost-effective, simplified deployment, easier maintenance, centralized backups
   - **Trade-offs**: Requires strict RLS policy enforcement, schema changes affect all tenants

**Q: 225. What are the pros and cons of your chosen approach?**

A: **[VAULT ANSWER]:**
   **PROS:**
   - Cost-effective infrastructure (single database cluster)
   - Simplified deployment and maintenance
   - Easy cross-tenant analytics for platform owner
   - Centralized backups and disaster recovery
   - Horizontal scaling with connection pooling
   - Schema updates deploy to all tenants simultaneously

   **CONS:**
   - RLS policy bugs could cause data leaks (mitigated with comprehensive testing)
   - All tenants affected by database downtime
   - Performance bottlenecks affect all tenants
   - Schema customization per tenant is complex
   - Requires strict tenant_id enforcement in all queries

**Q: 226. How do you ensure data isolation between universities?**

A: **[VAULT ANSWER]:** Multi-layered isolation strategy:
   - **Database Level**: PostgreSQL RLS policies automatically filter all queries by tenant_id
   - **Application Level**: JWT tokens contain tenant_id, enforced on every API request
   - **Authentication Layer**: 4-layer authentication system with Tenant ID + Department ID requirements
   - **Session Management**: Tenant-specific sessions prevent cross-contamination
   - **API Design**: All endpoints require tenant_id in request context
   - **Validation**: Comprehensive security testing planned before production

**Q: 227. How do you prevent data leaks or cross-contamination?**
